When we need Redux?

If you are experiencing the following symptoms you might need Redux:

A feeling that the state is spread out

Issues with updating state

State keeps changing but you don't know which or what

Solution

A single source of truth with reducers guarding state change

Also enhanced predictability with immutable data structures.

Core Concept of Redux

Store | our data store

Reducer | a function that takes state + action and produces a new state

Selector | selects a slice of state

Action | an intention of state change

Action creator | produces an intention that may include data

State

It is a single point of truth

Is read only

Everything is stored in a single tree

Action

Only way to update the state

Can have a payload

Are transitions between states

Last time, What is Redux/NgRx??

Redux is the pattern

NgRx is its Angular implementation

Reducer

A pure function that deals with state

It takes an action and the current state as a parameter

						
							Reducer(state, action) => new state

Always return a new state object; does not edit the current state

Reducers perform synchronous state changes

Let's see an example for Reducer!

Basically, it takes state and action as a parameter 
						
							export function reducer( state = initialState, action: Action ): State {
								switch (action.type) {
									case "LOAD_EVENTS":
										return { 
											...state, 
											events: action.payload 
										};
									default:
										return state;
								}
							}

Store

Store is in charge of storing the data

It holds the current state and all of the reducer

Store is the central object where everything happens

We use it to change the state

We use it to subscribe to state updates

Reducer & Store


Reducer

Pure function
Creates brand new state
Synchronous

Store

Holds the state
Used to dispatch actions
Used to get state updates

NgRx

Has its own store implementation
Uses RxJs so we can use observables and operators

What store really look like?

Typical store contain, just an object '{ }' 
						
							{
								counter: 'a value',
								user: [{ id: 1, name: 'Person' }],
								selectedUser: { id: 1, name: 'Person' }
							}

A simple Store!

Simplest form of store 
						
							class Store {
								constructor(initialState) { this.state = initialState; }
								dispatch(action) {
									this.state = calc(this.state, action);
								}
								calc(state, action) {
									return {
										counter: counterReducer(state.counter, action),
										users: usersReducer(state.users, action)
									}
								}
							}

Usage of Store

The way of using simple store 
						
							let store = new Store({ counter: 0, user: [] });
							store.dispatch({ type: 'INCREMENT' });
							// { counter: 1, users: [] }
							store.dispatch({
								type: 'ADD_USER',
								payload: { id: 1, name: 'Yoda' }
							});
							// { counter: 1, users: [{ id: 1, name: 'Yoda' }] }
							
							store.dispatch({
							type: 'REMOVE_User',
							payload: { id: 1 }
							});
							
							// { counter: 1, users: [] }
